# 115優高計畫書收件系統 — 部署步驟

完成後的系統架構：
- **老師填表** → GitHub Pages 靜態網頁（固定網址）
- **收資料** → Google Apps Script Web App → Google 試算表

---

## 第一步：建立 Google 試算表

1. 開啟 [Google 試算表](https://sheets.google.com) → 建立新試算表
2. 將試算表命名為：`115優高計畫書收件`
3. 複製網址列中的試算表 ID（在 `/d/` 和 `/edit` 之間的那段英數字串）
   - 例如：`https://docs.google.com/spreadsheets/d/`**`1aBcDeFgHiJkLmNoPqRsTuVwXyZ`**`/edit`
   - ID 就是：`1aBcDeFgHiJkLmNoPqRsTuVwXyZ`
4. 把這串 ID 先記下來備用

---

## 第二步：部署 Google Apps Script

1. 在剛才建立的試算表中，點上方選單 **延伸功能 → Apps Script**
2. 把預設的空白 `Code.gs` 內容全部刪除
3. 複製 `gas/Code.gs` 的全部內容貼上
4. 在第 6 行找到：
   ```javascript
   var SPREADSHEET_ID = ""; // ← 部署後填入試算表 ID
   ```
   把空字串改成你的試算表 ID：
   ```javascript
   var SPREADSHEET_ID = "1aBcDeFgHiJkLmNoPqRsTuVwXyZ";
   ```
5. 按 **儲存**（Ctrl+S 或 Command+S）
6. 在函式選單選擇 `initSheets`，按 **▶ 執行**
   - 第一次會要求授權，按「審查權限」→ 選你的 Google 帳號 → 進階 → 前往（不安全）→ 允許
   - 執行完成後，試算表會自動建立所有子工作表（A-1、A-2、B1-1…等）
7. 點右上角 **部署 → 新增部署作業**
8. 設定如下：
   - 類型：**網頁應用程式**
   - 說明：`115優高計畫收件`
   - 執行身分：**我**（你的 Google 帳號）
   - 誰可以存取：**所有人（含匿名使用者）**
9. 按 **部署** → 複製 **網頁應用程式 URL**（長得像 `https://script.google.com/macros/s/AKfycb.../exec`）
10. 把這個 URL 先記下來備用

---

## 第三步：更新 index.html 的 GAS URL

1. 用文字編輯器開啟 `index.html`
2. 搜尋這行（約在 JavaScript 最頂部）：
   ```javascript
   var GAS_URL = "https://script.google.com/macros/s/YOUR_SCRIPT_ID/exec";
   ```
3. 把 `YOUR_SCRIPT_ID` 那段換成你剛才複製的完整 URL：
   ```javascript
   var GAS_URL = "https://script.google.com/macros/s/AKfycbXXXXXX.../exec";
   ```
4. 儲存檔案

---

## 第四步：建立 GitHub 倉庫並上傳

1. 開啟 [GitHub](https://github.com) → 右上角 **+** → **New repository**
2. Repository name：`gaoyo-plan-115`
3. 設為 **Public**（GitHub Pages 免費方案需要 Public）
4. 不要勾選「Add a README file」（因為我們自己有檔案）
5. 按 **Create repository**

6. 在終端機執行：
   ```bash
   cd ~/gaoyo-plan-115
   git init
   git add .
   git commit -m "115優高計畫書收件系統初版"
   git branch -M main
   git remote add origin https://github.com/Leilei1221/gaoyo-plan-115.git
   git push -u origin main
   ```

7. 回到 GitHub 倉庫頁面 → **Settings** → 左側選 **Pages**
8. Source 選：**Deploy from a branch**
9. Branch 選：**main** / **/ (root)**
10. 按 **Save**
11. 等約 1～2 分鐘，重新整理頁面，上方會出現網址：
    `https://leilei1221.github.io/gaoyo-plan-115/`

---

## 第五步：測試完整流程

1. 開啟 `https://leilei1221.github.io/gaoyo-plan-115/`
2. 填寫一筆測試資料（隨便填個名字、選 A-2）
3. 按「確認送出」
4. 回到 Google 試算表 → 查看「A-2課綱課務推動」工作表，應該出現一列資料
5. 查看「📋所有提交」工作表，也應該有對應的記錄

---

## 修改後重新部署 GAS（若需要更新 Code.gs）

每次修改 `Code.gs` 後，要重新部署才會生效：
1. Apps Script 編輯器 → **部署 → 管理部署作業**
2. 點右側鉛筆圖示（編輯）
3. 版本選「**建立新版本**」→ 按儲存
4. URL 不會改變，不需要更新 `index.html`

## 常見問題

**Q：送出後顯示「送出失敗」**
- 確認 GAS_URL 已正確填入（不是 YOUR_SCRIPT_ID）
- 確認 GAS 是以「所有人（含匿名）」部署
- 確認 SPREADSHEET_ID 已填入且正確

**Q：試算表裡沒有資料**
- 確認你執行了 `initSheets()` 建立工作表
- 確認 SPREADSHEET_ID 是試算表 ID，不是整個網址

**Q：GitHub Pages 網址打開是 404**
- 等 2～3 分鐘讓 Pages 部署完成
- 確認 Settings → Pages 已設為 main branch / root
